# Profile surfaces

Use this file for personal and organization profile work.

## Personal profile checklist

- profile README present and intentionally structured
- bio is specific and audience-aware
- links and location fields are useful and current
- pinned repositories support the story the profile is telling
- widgets, if present, add signal rather than clutter

## Organization profile checklist

- `.github/profile/README.md` exists in the correct repository
- the org clearly explains what it builds
- key repositories are linked and easy to scan
- contributor, hiring, or contact paths are explicit when relevant
- branding and tone fit the organization rather than a personal profile template

## Typical outputs

- profile README draft or revision
- bio rewrite suggestions
- pinned-repository recommendations
- widget or badge cleanup suggestions
