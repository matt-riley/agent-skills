# Repository surfaces

Use this file for repository presentation and community-health work.

## Audit checklist

- README clarity and onboarding path
- repository description, topics, and other About metadata
- community health files
- issue and PR templates
- release and automation signals that affect trust
- whether the work is repo-local or belongs in an org-wide `.github` repository

## High-impact priorities

1. README and one-line description
2. community health files that match the repo's maturity
3. issue and PR templates
4. explicit manual metadata recommendations for About text, topics, website, or social preview image

## Common guardrails

- do not overwrite healthy existing files just to force a new template
- do not confuse repo-local templates with org defaults
- do not recommend boilerplate that ignores the stack or audience
